# 取得「別人可直接開、你電腦不用開」的連結

要讓別人隨時用連結查庫存，且你的電腦關機也能用，需要把系統放到 **Streamlit 雲端**（免費）。完成後你會得到一個網址，例如：`https://你的專案名稱.streamlit.app`。

---

## 一、事前準備

1. **GitHub 帳號**（沒有請先到 https://github.com 註冊）
2. **本機資料夾**裡要有：
   - `app.py`
   - `requirements.txt`
   - **`inventory.xlsx`**（要給別人查的庫存檔，務必放在同一資料夾）

---

## 二、把專案放到 GitHub

### 步驟 1：安裝 Git（若尚未安裝）

到 https://git-scm.com/download/win 下載並安裝。

### 步驟 2：在 GitHub 建立新倉庫

1. 登入 GitHub → 右上角 **+** → **New repository**
2. **Repository name** 填：`inventory-system`（或自訂英文名稱）
3. 選 **Private** 或 **Public**（Public 才能用免費 Streamlit 雲端）
4. **不要**勾選 "Add a README"
5. 按 **Create repository**

### 步驟 3：用指令上傳專案

在 **PowerShell** 或 **命令提示字元** 執行（路徑請改成你的實際路徑）：

```powershell
cd "c:\Users\georgina.wu\Desktop\Inventory system"

git init
git add app.py requirements.txt inventory.xlsx
git commit -m "庫存管理系統"
git branch -M main
git remote add origin https://github.com/junitawuu-byte/inventory-system.git
git push -u origin main
```

把上面 **`你的帳號/inventory-system`** 改成你在步驟 2 建立的倉庫網址（例如 `marywu/inventory-system`）。  
若 GitHub 要求登入，請用帳號密碼或 Personal Access Token。

---

## 三、在 Streamlit 雲端建立連結

1. 開啟 **https://share.streamlit.io**
2. 用 **Sign in with GitHub** 登入（同意授權）
3. 按 **New app**
4. 選擇：
   - **Repository**：選你剛建立的 `inventory-system`
   - **Branch**：`main`
   - **Main file path**：`app.py`
5. 按 **Deploy**
6. 等幾分鐘部署完成，畫面上會出現你的連結，例如：  
   **https://inventory-system-xxxxx.streamlit.app**

這個連結就是「別人可以直接開、你電腦不用開」的網址，可以複製傳給同事或存成我的最愛。

---

## 四、之後要更新庫存資料

1. 在電腦上更新 `inventory.xlsx` 後，到專案資料夾執行：

```powershell
cd "c:\Users\georgina.wu\Desktop\Inventory system"
git add inventory.xlsx
git commit -m "更新庫存"
git push
```

2. 到 **https://share.streamlit.io** → 點你的 app → 右上角 **Reboot app**，或等約 1 分鐘自動重啟，別人重新整理連結就會看到最新庫存。

---

## 常見問題

- **連結打不開？** 確認 GitHub 倉庫是 **Public**，且部署時沒有錯誤訊息。
- **別人看到的是舊資料？** 更新 `inventory.xlsx` 後要 `git add`、`commit`、`push`，再在 share.streamlit.io 重啟 app。
- **不想用 GitHub？** 若不想自己建倉庫，可考慮付費主機或請公司 IT 代為部署，但「別人可開、你電腦不用開」的連結一定需要放在某台 24 小時開著的伺服器或雲端上。
